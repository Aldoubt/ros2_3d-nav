# 常用命令

本文档记录当前推荐的 MID360 + FAST-LIVO2 + octomap 2D 地图保存 + ICP 重定位 + Nav2 导航流程。

主包：

```text
/home/yangxuan/ros2_ws/src/mid360_nav_demo
```

主思路：

```text
MID360 / Gazebo Livox
-> FAST-LIVO2 仅激光雷达里程计
-> /cloud_registered
-> octomap_server
-> /projected_map
-> map_saver_cli 保存 pgm/yaml
-> Ctrl+C 退出 FAST-LIVO2 保存 PCD
-> ICP 发布 map -> odom
-> Nav2 使用 map -> odom -> livox_frame 导航
```

### 3.2 保存 2D PGM/YAML

重要顺序：

```text
先保存 /projected_map
再 Ctrl+C 退出 FAST-LIVO2
```

因为 `/projected_map` 是在线发布的，关闭建图后就没有实时 2D 地图可保存。

保存：

```bash
cd /home/yangxuan/ros2_ws/src
source /opt/ros/humble/setup.bash
source /home/yangxuan/ros2_ws/src/install/setup.bash

ros2 launch mid360_nav_demo save_projected_map.launch.py \
  map_prefix:=/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1
```

输出：

```text
/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1.pgm
/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1.yaml
```

保存 launch 已处理两个关键参数：

```text
map_subscribe_transient_local:=false
save_map_timeout:=20.0
```

这是为了兼容 `octomap_server` 实时发布的 `/projected_map`，避免 QoS durability 不兼容。

验证：

```bash
ls -lh /home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1.*
cat /home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1.yaml
```

## 5. 参数怎么调

octomap 2D 投影主要看这些参数：

```text
pointcloud_min_z
pointcloud_max_z
occupancy_min_z
occupancy_max_z
filter_speckles
filter_ground
resolution
```

实车入口默认值在：

```text
mid360_nav_demo/launch/online_mapping_demo.launch.py
```

仿真入口默认值在：

```text
mid360_nav_demo/launch/sim_mapping_demo.launch.py
```

底层参数传给：

```text
mid360_nav_demo/launch/octomap_mapping.launch.py
```

备用记录配置：

```text
mid360_nav_demo/config/octomap_mapping.yaml
```

调参建议：

```text
地面黑色杂点多：略微提高 pointcloud_min_z 和 occupancy_min_z
低矮障碍识别不到：降低 pointcloud_min_z 和 occupancy_min_z
天花板/高处结构干扰：降低 pointcloud_max_z 和 occupancy_max_z
孤立小黑点多：保持 filter_speckles:=true
```

推荐起步值：

```bash
pointcloud_min_z:=0.05
occupancy_min_z:=0.08
pointcloud_max_z:=1.20
occupancy_max_z:=1.20
filter_speckles:=true
```

如果障碍物识别不到，先降低：

```bash
pointcloud_min_z:=0.03
occupancy_min_z:=0.05
```

如果地面误识别明显，先提高：

```bash
pointcloud_min_z:=0.10
occupancy_min_z:=0.12
```

先启动建图和 RViz：

```bash
ros2 launch mid360_nav_demo online_mapping_demo.launch.py
```

或仿真：

```bash

```

再开一个终端启动调参 GUI：

```bash
cd /home/yangxuan/ros2_ws/src
source /opt/ros/humble/setup.bash
source /home/yangxuan/ros2_ws/src/install/setup.bash

ros2 launch mid360_nav_demo octomap_tuning_gui.launch.py
```

GUI 功能：

```text
Refresh：读取 /octomap_server 当前参数
Apply Parameters：实时写入高度过滤参数
Apply + Reset Octomap：写参数后清空 octomap，适合重新观察调参效果
Reset Only：只清空当前 octomap
Save PGM/YAML：保存当前 /projected_map
```

注意：

```text
参数实时修改只影响后续进入 octomap 的点云。
旧地图里的黑点不会自动消失，所以调参时建议用 Apply + Reset Octomap。
RViz 负责看 /cloud_registered 和 /projected_map，GUI 负责调参和保存。
```

```bash
  global_map_pcd:=/home/yangxuan/ros2_ws/src/FAST-LIVO2-ROS2/Log/PCD/all_raw_points.pcd \
```

```bash
  launch_cmd_bridge:=true \

  bridge_publish_rate:=30.0
```

```bash
ros2 launch mid360_nav_demo icp_relocalizer.launch.py \
  global_map_pcd:=/home/yangxuan/ros2_ws/src/FAST-LIVO2-ROS2/Log/PCD/all_raw_points.pcd
```

说明：

```text
优先用 launch 启动 ICP，不建议直接 ros2 run。
当前环境的 LD_LIBRARY_PATH 里有 /opt/MVS/lib/64，直接 ros2 run 可能让 PCL 加载到 MVS 自带的旧 libusb。
icp_relocalizer.launch.py 已经用 LD_PRELOAD 指定系统 libusb，避免 libusb_set_option 符号错误。
```

仿真里推荐用专门的重定位入口：

```bash
source /home/yangxuan/ros2_ws_test/PB_RMSimulation/install/setup.bash
source /home/yangxuan/ros2_ws/src/install/setup.bash

ros2 launch mid360_nav_demo sim_relocalization_demo.launch.py \
  map:=/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/sim_test.yaml \
  global_map_pcd:=/home/yangxuan/ros2_ws/src/FAST-LIVO2-ROS2/Log/PCD/all_raw_points.pcd
```

这个入口会启动：

```text
Gazebo + MID360
FAST-LIVO2 仿真重定位配置，pcd_save_en: false，不会覆盖原 PCD
map_server 发布 /map
icp_relocalizer_node 发布 map -> odom
RViz 固定坐标系 map，并带 2D Pose Estimate
sim_drive_gui 发布 /cmd_vel_chassis
```

RViz 里如果没有看到 `2D Pose Estimate`，确认使用的是：

```text
mid360_nav_demo/rviz/mid360_sim_relocalization.rviz
```

或者手动添加工具：

```text
Panels/Tools -> Add Tool -> rviz_default_plugins/SetInitialPose
Topic: /initialpose
```

重定位 RViz 观察顺序：

```text
ICP 成功前：还没有 map -> odom，实时 /cloud_registered 和 /path 更适合在 odom 下看。
发布 2D Pose Estimate 后：ICP 成功会发布 map -> odom，这时切到 map 可以同时看 /map、TF 和对齐后的点云。
```

检查 map_server 是否正常发布 PGM：

```bash
ros2 lifecycle get /map_server
ros2 topic info /map -v
ros2 topic echo /map --once --field info
```

检查 FAST-LIVO2 里程计轨迹：

```bash
ros2 topic hz /path
ros2 topic echo /aft_mapped_to_init --once
```

如果 RViz 里 `/map` 看不到，优先确认：

```text
map 参数是否指向存在的 yaml
map_server lifecycle 是否 active
RViz Map display 话题是否是 /map
Fixed Frame 可以手动输入 map，不一定必须从下拉框里选择
```

RViz 里：

```text
Fixed Frame: map
先点 2D Pose Estimate，触发 ICP 重定位
ICP 成功后再发 Nav2 Goal
```

## 8. 常见问题

### 8.1 `/livox/lidar` 同名多类型

仿真里如果看到：

```text
/livox/lidar contains more than one type
```

优先查详细端点：

```bash
ros2 topic info /livox/lidar -v
```

FAST-LIVO2 正确订阅仿真雷达时应看到：

```text
fastlivo_mapping subscribes livox_ros_driver2/msg/CustomMsg
```

### 8.2 `/projected_map` 保存失败

如果出现 QoS durability 不兼容，使用当前保存 launch：

```bash
ros2 launch mid360_nav_demo save_projected_map.launch.py \
  map_prefix:=/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1
```

如果只是超时，拉长等待：

```bash
ros2 launch mid360_nav_demo save_projected_map.launch.py \
  map_prefix:=/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1 \
  save_map_timeout:=60.0
```

### 8.3 PGM 出现黑色地面杂点

这不是 RViz 显示坐标系单独造成的。如果黑点已经保存进 PGM，说明 `/projected_map` 已经把点投成 occupied。

调参方向：

```bash
pointcloud_min_z:=0.08
occupancy_min_z:=0.10
filter_speckles:=true
```

如果障碍物消失，降低高度阈值：

```bash
pointcloud_min_z:=0.03
occupancy_min_z:=0.05
```

### 8.4 conda 环境影响

建图、导航、构建 ROS2 包时建议：

```bash
conda deactivate
```

`(.venv-jie3d)` 主要给 GUI/Open3D 工具用，不建议混在 FAST-LIVO2/Nav2 主流程里。

#### 10.6 现场异常快速定位

雷达没有数据：

```bash

```

FAST-LIVO2 没有点云：

```bash

ros2 topic echo /aft_mapped_to_init --once
```

2D 地图没有发布：

```bash
ros2 topic info /projected_map -v
ros2 topic echo /projected_map --once --field info
```

保存地图失败：

```bash
ros2 launch mid360_nav_demo save_projected_map.launch.py \
  map_prefix:=/home/yangxuan/ros2_ws/src/mid360_nav_demo/maps/site1 \
  save_map_timeout:=60.0
```

RViz 看不到保存的 `/map`：

```bash
ros2 lifecycle get /map_server
ros2 topic info /map -v
ros2 topic echo /map --once --qos-durability transient_local --field info
```

ICP 不成功：

```bash
ros2 topic echo /relocalization/status --once
ros2 run tf2_ros tf2_echo map odom
```

如果 ICP 初始误差太大：

```text
重新在 RViz 发布更接近真实位置和朝向的 2D Pose Estimate
降低车速，保持车辆静止几秒再重定位
必要时增大 icp_relocalizer.yaml 里的 max_correspondence_distance
```

电脑明显卡顿：

```bash
free -h
ps -eo pid,%cpu,%mem,rss,comm,args --sort=-%cpu | head -n 20
```

降低负载方法：

```text
RViz 关闭 /cloud_registered 或 /relocalization/aligned_cloud 显示
导航时不用建图入口，使用 online_nav_demo.launch.py
确认没有重复开多个 Gazebo/RViz/FAST-LIVO2 launch
```

#### 10.7 现场最小安全原则

```text
先离线测试 ICP，再连接底盘
先低速短距离发目标点，再扩大距离
Nav2 Goal 发出前确认 map -> odom -> livox_frame 正常
任何时候车体行为异常，先停底盘或 Ctrl+C 导航 launch
建图保存时先保存 PGM/YAML，再 Ctrl+C 保存 PCD
```
